## 2. Page Location (Nav Link â†’ App Config)

**Problem:** Clicking "Sitemap & Tree Manager" in the Contentful Apps nav showed a blank "Open this app within Contentful." screen.

**Root cause:** App had no `page` location handler. Contentful loads the page location when the nav link is clicked, but the app fell through to the fallback message.

**Fix:**
- Added `{ "location": "page" }` to `contentful-app-manifest.json`
- Added `PageLocation` component in `app-with-sdk.tsx` that calls `sdk.navigator.openAppConfig()` on mount â€” immediately redirects to App Config

**Note:** Requires "Page" location to be enabled in the Contentful App Definition with "Show app in main navigation" checked. Link path should be `/`.

**Commit:** `f589327`

## 4. Slug Field Widget Auto-Assignment

**User feedback:** After enabling a content type and selecting a slug field in App Config, the slug field's appearance didn't automatically switch to use the app widget â€” editors still saw the native Contentful slug widget until they manually changed it.

**Fix:** On every App Config save, for each enabled CT, directly call the CMA editor interface API to set `widgetId: sdk.ids.app` / `widgetNamespace: "app"` on the selected slug field. Follows the same pattern as `applyEditorInterfaceSettings()` for the Sitemap CT.

**On disable:** When a CT is toggled off and saved, the slug field widget is reverted to `builtin/singleLine`.

**Bug found and fixed (same session):** Initial revert logic used `currentState.EditorInterface` to detect previously-managed CTs. This was unreliable â€” if `slugFieldId` wasn't set when the CT was first enabled, the CT never appeared in `targetState.EditorInterface`, and the revert would never trigger. Fixed by:
1. Using `sdk.app.getParameters()` to get the previously-saved `enabledContentTypes` directly
2. Removing the `...currentState?.EditorInterface` spread from the `targetState` return â€” it was preserving disabled CTs in targetState, preventing Contentful's own cleanup mechanism from working

**Commits:** `ce72d5c`, `1202c9a`, `9694e3f`
